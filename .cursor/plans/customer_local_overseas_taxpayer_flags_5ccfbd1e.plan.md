# Customer residency + taxpayer flags

## Decisions locked in
- **Residency modeling**: enum `CustomerResidency` with values `LOCAL | OVERSEAS`.
- **Taxpayer rule**: when `hasTaxpayerId` is `false`, we **force** `taxpayerId = null` in persistence.

## Where changes will land
- Prisma schema: `prisma/schema.prisma`
- Customer server actions: `app/(app)/customers/actions.ts`
- Customers page query/select mapping: `app/(app)/customers/page.tsx`
- Customers UI form/table: `app/(app)/customers/CustomersClient.tsx`
- DB migration: generated by `prisma migrate`.

## Schema changes (Prisma)
- Add a new enum:
  - `enum CustomerResidency { LOCAL OVERSEAS }`
- Update `model Customer`:
  - Add `residency CustomerResidency @default(LOCAL)`
  - Add `hasTaxpayerId Boolean @default(false)`
  - Keep `taxpayerId String?` (still optional)
- Add any helpful index only if we will query by residency later; otherwise skip.

## Server actions (create/update)
- Parse `residency` from `FormData` with safe fallback to `LOCAL`.
- Parse `hasTaxpayerId` from `FormData`:
  - Treat checked checkbox as true (presence of `"on"`), otherwise false.
- Enforce rule:
  - If `hasTaxpayerId === false`, set `taxpayerId = null` regardless of what was submitted.
  - If true, accept `taxpayerId` trimmed, else null.
- Update `prisma.customer.create()` and `.update()` payloads to include `residency` and `hasTaxpayerId`.

## Customers page query
- Extend `select` in `prisma.customer.findMany()` to include:
  - `residency`
  - `hasTaxpayerId`
- Include these fields in the `customers` prop mapping passed to `CustomersClient`.

## Customers UI (`CustomersClient.tsx`)
- Extend local `CustomerRow` type to include `residency` and `hasTaxpayerId`.
- Extend local `form` state:
  - `residency` (default `LOCAL`)
  - `hasTaxpayerId` (already added in UI; we’ll make it persist)
- Add a UI control for residency (select or radio):
  - Labeled “Customer location” (or your preferred text)
  - Options: Local, Overseas
  - Name it `residency` so actions can read it.
- Ensure edit mode initializes:
  - `form.residency` from row
  - `form.hasTaxpayerId` from row (not inferred only from `taxpayerId`)
- Keep taxpayer UI behavior:
  - Checkbox toggles visibility of the taxpayer input.
  - When unchecked, also clear `form.taxpayerId` and submit hidden `taxpayerId=""` (so update clears).
  - Additionally submit the checkbox value via its `name="hasTaxpayerId"`.

## Verification
- Run `prisma migrate dev` to generate/apply migration.
- Run `npm run build` to confirm TypeScript + Next build.
- Manually verify:
  - Create local + overseas customers.
  - Toggle `hasTaxpayerId` off after saving a number and confirm it clears in list + on next edit.
